---

## 2 근거

### 2.1 keepalived 라이선스

- keepalived = **GPL-2.0-or-later** (`COPYING` = GNU GPL v2, 헤더에 "or any later version").
- GPL은 카피레프트: GPL 코드를 포함·파생한 결과물도 GPL로 공개해야 함 (전염성).

### 2.2 왜 GPL이 부담이 아닌가

처음 구상은 MIT였으나, **keepalived 소스를 코드 수준까지 최대 활용**하기로 하면서 GPL이 자연스러운 선택이 됨.

| GPL의 제약 | 본 프로젝트에 미치는 영향 |
|------------|--------------------------|
| 소스 공개 의무 | 없음 — 어차피 오픈소스 무료 공개 |
| 상업화 제약 | 없음 — 수익 목표 없음 |
| 영구 GPL 고정 | 수용 — 라이선스 변경 계획 없음 |
| 기여자 GPL 동의 | 수용 — 오픈소스 전제 |

→ GPL의 모든 trade-off가 본 프로젝트엔 비용이 아님.

### 2.3 GPL 채택의 실익

- keepalived 소스의 **타입·검증 로직을 직접 참조·이식 가능**.
- 조사에서 "값 타입은 핸들러 로직 안에 있어 반자동 추출"이라던 부분의 정확도가 올라감.
- "사실만 추출" 같은 경계 신경 쓸 필요 없음 → 개발 단순화.

## 3 keepalived 소스 활용 범위

GPL 수용 전제로, 다음을 모두 허용한다.

| 활용 | 허용 |
|------|------|
| 지시어·중첩 구조 추출 | O |
| 값 타입 추출 (핸들러 로직 참조) | O |
| 검증 로직 일부 이식 (범위 체크 등) | O |
| keepalived 파서 동작 모방 | O |

---

## 4 의무 사항 (Attribution)

GPL 준수를 위해 반드시:

- [ ] 레포 루트에 `LICENSE` = GPL-2.0 전문
- [ ] keepalived 출처·저작권 명시 (NOTICE 또는 README)
- [ ] keepalived 소스에서 유래한 부분(스키마, 이식 로직) 출처 주석
- [ ] `package.json` `license` 필드 = `"GPL-2.0-or-later"`
