# ADR-0009: 부분 스키마 커버리지에서 미지시어 침묵 정책

- 상태: Accepted
- 날짜: 2026-06-29

## Context

ADR-0008로 스키마를 수작업 시드(현재 15블록)로 시작한다. keepalived 전체 키워드의
일부만 커버한다. 검증 구문층이 "스키마에 없으면 미지시어 error"를 단순 적용하면,
**아직 시드 안 한 정상 키워드까지 빨간 줄**이 된다 → 대량 오탐.

validation §1.1: "오탐 1건이 신뢰 10건을 깎는다."

## Decision

미지시어/미지블록 판정을 **부모 블록이 `complete: true` 일 때만** 수행한다.

- 블록의 `complete` 플래그 = "지시어/자식블록 목록이 완전함" 명시.
- 시드 스키마는 부분적이므로 어떤 블록도 `complete` 가 아님 → 미지시어/미지블록 전부 침묵.
- 한 블록의 지시어·자식블록을 전수 채운 뒤에야 `complete: true` 로 표시 → 그때부터 검출.
- `SYNTAX_INVALID_PARENT`(알려진 블록의 잘못된 위치)와 `SYNTAX_WRONG_KIND`
  (블록을 `{}` 없이 인라인)는 확실하므로 `complete` 와 무관하게 항상 검사.

> 정정 이력: 초기 구현은 "부모가 스키마에 있으면 자식 목록을 완전하다 간주"였다.
> 그러나 시드는 부분적이라, 정의된 블록(global_defs 등)의 미수록 정상 지시어가 대량
> 오탐됐다(리뷰 M2/M3). `complete` 게이트로 교정 — 완전성을 명시할 때만 단정.

## Consequences

- (+) 시드 단계에서도 오탐 0에 수렴 → 신뢰성 우선 충족.
- (+) 스키마 커버리지가 늘수록 검출 범위가 자연히 확대됨(코드 변경 없이).
- (−) 시드 안 된 블록 내부의 진짜 오타는 못 잡음 → 커버리지 확대로 점진 해소.
- 함의: 스키마 완성도가 곧 검증 강도. 흔한 블록부터 directives 를 채우는 게 우선.

## Alternatives

- 무조건 미지시어 error: 대량 오탐. 기각.
- 모든 미지시어 warning: 정상 키워드에도 노란 줄 → 여전히 노이즈. 기각.
